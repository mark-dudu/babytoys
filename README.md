# BabyToys

Windows 儿童模式防误触工具。一键进入儿童模式后，程序会覆盖所有显示器、短暂显示系统壁纸或自定义图片，随后转为黑屏，并尽量拦截常规键盘鼠标输入。倒计时结束后会优先请求系统睡眠。

当前开发版本为 v1.2：在 v1.1 日常使用能力基础上，加入自动化测试、配置恢复、日志轮转、单实例保护以及可重复的 CI/发布流程；仍暂不提供安装器、自动更新或安全锁屏承诺。

## 目标边界

第一版目标是防孩子乱拍导致误操作、降低屏幕吸引力、保留家长可恢复路径。它不是安全锁屏软件，也不承诺拦截 `Ctrl+Alt+Del`、硬件电源键、UAC 安全桌面或所有系统级快捷键。

家长解锁方式：`Ctrl + Alt + U` 长按 3 秒后松开。

进入儿童模式前有 3 秒确认倒计时，可点击取消或按 `Esc` 取消。

## 技术栈

- C# / .NET 10 / WPF
- Windows 10/11
- WPF 原生控件
- Win32 P/Invoke 封装系统能力
- 无第三方依赖

## 运行

需要在 Windows 上安装 .NET 10 SDK。

```powershell
dotnet build
dotnet run
```

执行自动化测试：

```powershell
dotnet test --project BabyToys.Tests/BabyToys.Tests.csproj
```

发布单文件版本：

```powershell
dotnet publish -c Release -r win-x64 --self-contained true /p:PublishSingleFile=true /p:IncludeNativeLibrariesForSelfExtract=true
```

输出文件位于 `bin/Release/net10.0-windows10.0.17763.0/win-x64/publish/`。

生成带版本号的 ZIP 和 SHA-256 校验文件：

```powershell
./scripts/Publish-Release.ps1
```

输出位于 `artifacts/`。`main` 分支和 Pull Request 会在 Windows GitHub Actions 中执行构建、测试和打包冒烟验证；推送与项目版本匹配的标签后会自动创建 GitHub Release。

## 当前功能

- 主窗口设置图片来源和时间限制，支持小数分钟；时间会限制在 0.5 到 240 分钟之间。
- 默认使用系统壁纸，也支持自定义图片。
- 进入前有 3 秒确认倒计时，可点击取消或按 `Esc` 取消。
- 儿童模式覆盖所有显示器。
- 儿童模式遮罩窗口会隐藏鼠标箭头，退出后由系统自动恢复。
- 图片阶段叠加深色遮罩，10 秒后淡入纯黑屏。
- 使用 low-level keyboard hook 和 mouse hook 尽量拦截常规输入。
- 超时后释放钩子并请求系统睡眠；睡眠请求成功后结束儿童模式，睡眠请求失败时保持黑屏并允许解锁。
- 配置保存到 `%LocalAppData%/BabyToys/settings.json`。
- 配置使用版本化结构和原子写入；文件损坏时会备份原文件并恢复安全默认值。
- 日志写入 `%LocalAppData%/BabyToys/logs`，限制文件大小并只保留最近 14 个文件；只记录进入、退出、超时和错误，不记录具体按键内容。
- 同一 Windows 会话只允许运行一个 BabyToys 实例，避免托盘、快捷键和设置互相冲突。
- 支持保存、应用和删除多组儿童模式预设。
- 支持 `Ctrl + Alt + B` 全局快捷键启动，仍会经过 3 秒确认倒计时。
- 关闭主窗口后继续在系统托盘运行，可从托盘打开、启动儿童模式或安全退出。
- 可选择登录 Windows 后自动启动到托盘。
- 启动时会核对实际 Windows 启动项与保存设置，并自动清理不一致的残留启动项。
- 若 Windows 调用了缓存的 `--minimized` 启动命令，而设置中未启用开机自启，应用会立即退出。
- 正常儿童模式会维护活动标记；若进程异常终止，下次启动会提示并记录诊断日志。
- 通过 `Ctrl + Alt + Del` 打开并切换到任务管理器时，会立即释放输入拦截和遮罩，保留系统级紧急恢复路径。
- 诊断窗口汇总版本、系统、显示器、配置路径和可唤醒设备，不采集具体按键内容。
- 通过 `powercfg` 只读检查允许唤醒电脑的设备，并提供关闭和恢复唤醒权限的命令指引。

## 已知限制

- 不能阻止 `Ctrl + Alt + Del`、UAC 安全桌面、硬件电源键或所有系统级快捷键。
- 输入拦截依赖 Windows low-level hook，可能受安全软件、权限或系统策略影响。
- 睡眠请求可能被系统电源策略、外接设备或其他应用阻止。
- 当前版本没有安装器、密码解锁或远程解锁。
- 全局快捷键可能被其他程序占用；占用时主窗口会显示提示。

## 手动验证清单

发布或打包前建议在 Windows 10/11 实机验证：

- `dotnet build` 能成功。
- 主窗口可保存图片来源、时间限制和倒计时显示设置。
- 预设可以保存、应用、覆盖和删除。
- `Ctrl + Alt + B` 可以唤起窗口并开始 3 秒确认倒计时。
- 关闭主窗口后托盘图标仍然可用，托盘“退出”可以完整结束程序。
- 开机自启选项可以正确写入和移除当前用户启动项。
- 主窗口“诊断”可以显示系统信息和当前允许唤醒电脑的设备。
- 复制诊断报告不会包含具体按键或输入内容。
- 正常解锁或退出后不会遗留 `active-session.json`；模拟异常终止后，下次启动会显示恢复提示。
- 确认倒计时期间点击取消和按 `Esc` 都能取消。
- 儿童模式覆盖所有显示器。
- 儿童模式期间所有遮罩屏幕均不显示鼠标箭头，解锁或退出后鼠标箭头恢复。
- 常规键盘和鼠标输入被拦截。
- `Ctrl + Alt + U` 长按 3 秒后松开可以解锁，按键不会泄漏到恢复后的窗口或输入法。
- 长按解锁时所有遮罩屏幕显示剩余秒数和进度条，达到 3 秒后提示“松开即可解锁”。
- `Ctrl + Alt + Del` 打开任务管理器后，儿童模式会自动结束且任务管理器可见、可操作。
- 超时后优先请求系统睡眠。
- 退出或解锁后窗口、计时器和输入钩子被清理，系统恢复正常输入。
